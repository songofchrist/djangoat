Overview

We will be using the following constants in the tag examples below:
  • DATE_FORMAT: "F jS, Y"
  • TIME_FORMAT: "F jS, Y g:i a"
  • NUM_DICT: {'one': 1, 'two': 2, 'three': 3}


Filters

data

{% verbatim %}{{ KEY|data:ARG }}{% endverbatim %}

Here we have the filter version of the `_data tag`_. This filter uses KEY to pull from Djangoat's DATA dict, accepting at most one argument, and outputs the result. Consider the examples below in light of following functions:

  • DATA['now']: returns the current time
  • DATA['cube_func']: cubes ARG
  • DATA['range_func']: creates a range with a length of ARG

{% verbatim %}{{ 'now'|data }}{% endverbatim %}
 {{ 'now'|data }} 
{% verbatim %}{{ 'now'|data|date:DATE_FORMAT }}{% endverbatim %}
 {{ 'now'|data|date:DATE_FORMAT }} 
{% verbatim %}{{ 'cube_func'|data:2 }}{% endverbatim %}
 {{ 'cube_func'|data:2 }} 
{% verbatim %}{% for r in 'range_func'|data:5 %}{{ r }}! {% endfor %}{% endverbatim %}
 {% for r in 'range_func'|data:5 %}{{ r }}! {% endfor %}

Tags

data

{% verbatim %}{% data KEY ARG1 ARG2 ARG3 . . . %}{% endverbatim %}

Here we have the tag version of `_data filter`_. This tag uses KEY to pull from Djangoat's DATA dict and is required when accessing a function that requires more than a single argument. Consider the examples below in light of following functions:

  • DATA['now']: returns the current time
  • DATA['power_func']: takes ARG1 to the power of ARG2

{% verbatim %}{% data 'now' %}{% endverbatim %}
 {% data 'now' %} 
{% verbatim %}{% data 'now' as now %}{{ now|date:DATE_FORMAT }}{% endverbatim %}
 {% data 'now' as now %}{{ now|date:DATE_FORMAT }} 
{% verbatim %}{% data 'now>' %}{{ now|date:DATE_FORMAT }}{% endverbatim %}
 {% data 'now>' %}{{ now|date:DATE_FORMAT }} 
{% verbatim %}{% data 'power_func' 3 4 %}{% endverbatim %}
 {% data 'power_func' 3 4 %}

cache, usercache, sitecache, usersitecache

{% verbatim %} {% cache TIME NAME ARG1 ARG2 . . . %}Cached Content{% endcache %}
{% usercache TIME NAME ARG1 ARG2 . . . %}Cached Authenticated User Content{% endusercache %}
{% sitecache TIME NAME ARG1 ARG2 . . . %}Cached Current Site Content{% endsitecache %}
{% usersitecache TIME NAME ARG1 ARG2 . . . %}Cached Authenticated User Content For The Current Site{% endusersitecache %}
{% endverbatim %}

Here we see calls to the cache tag and its variants with the resulting cached content. To clear these individually or all at once, visit the CacheFrag admin area, select the fragments to clear, and select "Clear selected Cache Fragments" from the dropdown. They should update on the next page load.
